# AGENTS.md

Repo-specific guidance for AI coding agents. Stack: Vite + React 19 + TanStack Router + TanStack Query + Shadcn UI + Tailwind v4 + Zod + react-hook-form, with optional Clerk auth and Supabase backend. `package.json` name is `eburon-ai`.

## Commands

Package manager is **pnpm** (CI + README use it; `pnpm-lock.yaml` is canonical). A `package-lock.json` also exists — ignore it and never run `npm install`.

```bash
pnpm install
pnpm dev                    # Vite dev server (regenerates route tree on start)
pnpm build                  # tsc -b (typecheck) THEN vite build — no separate typecheck script exists
pnpm lint                   # eslint .
pnpm format                 # prettier --write .
pnpm format:check           # prettier --check .  (CI uses this, not `format`)
pnpm knip                   # unused files/deps
pnpm test <path>            # single file, headless browser
pnpm test:watch <path>
pnpm test:ui                # vitest UI
pnpm test:coverage
```

Before the first test run on a machine: `pnpm test:browser:install` (Playwright chromium). `--browser.headless` is already baked into the scripts — don't add it.

CI order (`.github/workflows/ci.yml`): `lint → format:check → test:browser:install → test → build` (knip step is commented out). Local pre-submit per CONTRIBUTING: `pnpm lint && pnpm format && pnpm knip && pnpm build`.

## Tests run in a real browser (not jsdom)

Vitest is in **browser mode** via `@vitest/browser-playwright` (chromium). Component tests import:

```ts
import { render } from 'vitest-browser-react'
import { userEvent } from 'vitest/browser'
```

Do **not** use `@testing-library/react` or `@testing-library/user-event` — not installed. Assertions use `await expect.element(...)` (vitest-browser API). Tests are colocated: `*.test.tsx` (components) / `*.test.ts` (libs). Shared helpers in `src/test-utils/` (cookie clearing, TanStack Table mock) — prefer them over rolling your own.

Coverage excludes `src/components/ui/**`, `src/routes/**`, `src/routeTree.gen.ts`, `src/test-utils/**`, `src/assets/**` — write tests under `src/features/**`, `src/components/**` (non-ui), `src/lib/**`, `src/hooks/**`, `src/context/**`.

## Generated / vendored — do not hand-edit

- `src/routeTree.gen.ts` — generated by `@tanstack/router-plugin/vite` (`autoCodeSplitting`) on dev/build. Carries `@ts-nocheck` + `/* eslint-disable */`; excluded from prettier + coverage. Routes are file-based under `src/routes/`.
- `src/components/ui/**` — Shadcn UI. Ignored by eslint, knip, coverage. Some are customized for RTL/behavior — do **not** blindly overwrite with `npx shadcn add`. Modified: `scroll-area`, `sonner`, `separator`. RTL-updated: `alert-dialog`, `calendar`, `command`, `dialog`, `dropdown-menu`, `select`, `table`, `sheet`, `sidebar`, `switch`.
- `.tanstack/tmp/` — router plugin temp dir.

## Architecture

- Entry `src/main.tsx`: provider stack `ClerkProvider → QueryClientProvider → ThemeProvider → FontProvider → DirectionProvider → RouterProvider`. Router is built from the generated `routeTree` with context `{ queryClient }` and type-registered via `declare module '@tanstack/react-router'`. `QueryClient` retries are disabled in dev; 500s navigate to `/500` in prod.
- Routes (`src/routes/`, file-based): `__root.tsx` is root; `_authenticated/` is the authed layout (`AuthenticatedLayout`); `(auth)`/`(errors)` are pathless route groups; `clerk/` is an optional Clerk demo.
- Features (`src/features/<domain>/`): `apps`, `chats`, `dashboard`, `database`, `errors`, `issues`, `models`, `settings`, `tasks`, `users` — typically `components/`, `data/`, `index.tsx`.
- Lib (`src/lib/`): supabase client + queries/apps, `users-db.ts`, `handle-server-error.ts`, `cookies.ts`, `utils.ts`.
- Path alias `@/*` → `src/*` (tsconfig + vite).
- Tailwind v4 — CSS config in `src/styles/index.css` + `theme.css`; **no `tailwind.config.js`**. Uses `@tailwindcss/vite`.

## Lint / format rules that differ from defaults

- Prettier: **no semicolons**, single quotes (incl. JSX), 2-space, printWidth 80, trailingComma `es5`, LF.
- Import order is enforced by `@trivago/prettier-plugin-sort-imports` — `pnpm format` will rewrite. Order: `react` → named packages (`zod`, `axios`, `date-fns`, `react-hook-form`, `@radix-ui/*`, `@tanstack/*`) → third-party → `@/` aliases in a fixed order (`assets`, `api`, `stores`, `lib`, `utils`, `constants`, `context`, `hooks`, `components/layouts`, `components/ui`, `components/errors`, `components`, `features`) → relative. Match it or format will move your imports.
- Tailwind class sorting via `prettier-plugin-tailwindcss` (functions: `cn`, `clsx`).
- ESLint: `no-console: error` — for intentional console, add `// eslint-disable-next-line no-console` (see `main.tsx`, `src/lib/supabase.ts`).
- `@typescript-eslint/consistent-type-imports: error` — use `import type` or inline `import { type X }`.
- `@typescript-eslint/no-unused-vars` — prefix unused args/catch/destructure with `_`.
- `no-duplicate-imports: error`.

## Env / backend

Copy `.env.example` → `.env` (gitignored). Vars:

- `VITE_CLERK_PUBLISHABLE_KEY` — Clerk is **optional**; the app runs without it (shows a setup page at `/clerk`). Clerk integration lives entirely under `src/routes/clerk/` and can be removed along with the `@clerk/react` dep without affecting the rest of the app.
- `VITE_SUPABASE_URL`, `VITE_SUPABASE_ANON_KEY` — client in `src/lib/supabase.ts` (also accepts `NEXT_PUBLIC_*` fallbacks). The DB schema in `supabase/schema.sql` must be applied manually via the Supabase SQL editor.

## Commits / PRs

- Conventional commits (commitizen `cz.yaml` + history: `feat:`, `fix:`, `chore:`, `refactor:`, `build(deps):`). `cz` bumps update `CHANGELOG.md` and tag `v$version`.
- Default branch is `main` (CI runs here). New features require a prior issue/discussion per `.github/CONTRIBUTING.md`; PRs follow `.github/PULL_REQUEST_TEMPLATE.md`.
- Deploy: Netlify SPA — `netlify.toml` redirects `/*` to `/index.html`. Vercel SPA — `vercel.json` rewrites all routes to `/index.html`. Both exist; either platform deploys with `pnpm build` → `dist/`.
