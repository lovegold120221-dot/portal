# AGENTS.md

Stack: Vite + React 19 + TanStack Router + TanStack Query + Shadcn UI + Tailwind v4 + Zod + react-hook-form + Zustand. `package.json` name: `eburon-ai`.

## Commands

Package manager is **pnpm**. A `package-lock.json` exists — ignore it, never run `npm install`.

```bash
pnpm install
pnpm dev                      # Vite dev server
pnpm build                    # tsc -b (typecheck) THEN vite build
pnpm lint                     # eslint .
pnpm format                   # prettier --write .
pnpm format:check             # prettier --check .   (CI uses this)
pnpm knip                     # unused files/deps
pnpm test:browser:install       # PLAYWRIGHT: run once per machine (installs chromium + deps)
pnpm test <path>              # vitest run --browser.headless
pnpm test:ui                  # vitest --ui --browser.headless
pnpm test:coverage
```

CI order (`.github/workflows/ci.yml`): `lint → format:check → test:browser:install → test → build`. knip step is commented out.

Tests run in a **real browser** (chromium via Playwright, `@vitest/browser-playwright`), NOT jsdom.
Coverage excludes: `src/components/ui/**`, `src/routes/**`, `src/routeTree.gen.ts`, `src/test-utils/**`, `src/assets/**`.

## Tests

- Imports: `render` from `vitest-browser-react`, `userEvent` from `vitest/browser`.
- Assertions: `await expect.element(...)` (vitest-browser API).
- Do **not** import from `@testing-library/react` or `@testing-library/user-event` — not installed.
- Colocated: `*.test.tsx` for components, `*.test.ts` for libs. Shared helpers in `src/test-utils/`.

## Generated / vendored — do not hand-edit

- `src/routeTree.gen.ts` — generated by `@tanstack/router-plugin/vite` on every dev/build (`autoCodeSplitting: true`). Has `@ts-nocheck` + `/* eslint-disable */`.
- `src/components/ui/**` — Shadcn UI with RTL/behavior mods. Do **not** blind-overwrite with `npx shadcn add`. Modified: `scroll-area`, `sonner`, `separator`. RTL-updated: `alert-dialog`, `calendar`, `command`, `dialog`, `dropdown-menu`, `select`, `table`, `sheet`, `sidebar`, `switch`. All others are standard.
- `.tanstack/tmp/` — router plugin temp dir.

## Architecture

**Entry (`src/main.tsx`):** Provider stack: `ClerkProvider → QueryClientProvider → ThemeProvider → FontProvider → DirectionProvider → RouterProvider`. Router built from generated `routeTree` with context `{ queryClient }`. 500 errors navigate to `/500` in prod.

**Routes (`src/routes/`, file-based):** `__root.tsx` is root. `_authenticated/` is the authed layout (`AuthenticatedLayout`). `(auth)`/`(errors)` are pathless route groups. `clerk/` is an optional Clerk demo route.

**Features:** `src/features/<domain>/` — `apps`, `chats`, `dashboard`, `database`, `errors`, `issues`, `models`, `settings`, `tasks`, `users`. Structure: `components/`, `data/`, `index.tsx`.

**Lib:** `src/lib/` — supabase client + queries/apps, `users-db.ts`, `handle-server-error.ts`, `cookies.ts`, `utils.ts`.

**Alias:** `@/*` → `src/*` (tsconfig + vite).

## Prettier / ESLint rules

- Prettier config: `.prettierrc`. **No semicolons**, single quotes (incl. JSX `jsxSingleQuote`), 2-space, printWidth 80, trailingComma `es5`, LF line endings.
- **Import order** (managed by `@trivago/prettier-plugin-sort-imports`, `pnpm format` rewrites): `react` → `@vitejs/*` → `globals` → `zod` → `axios` → `date-fns` → `react-hook-form` → `use-intl` → `@radix-ui/*` → `@hookform/resolvers/zod` → `@tanstack/react-query` → `@tanstack/react-router` → `@tanstack/react-table` → `THIRD_PARTY_MODULES` → `@/*` aliases: `assets`, `api`, `stores`, `lib`, `utils`, `constants`, `context`, `hooks`, `components/layouts`, `components/ui`, `components/errors`, `components`, `features` → relative. Match this or format will move your imports.
- Tailwind class sorting via `prettier-plugin-tailwindcss`. Configured for functions `cn`, `clsx` (via `tailwindFunctions` in `.prettierrc`).
- ESLint: `no-console: error` (add `// eslint-disable-next-line no-console` for intentional use — see `main.tsx` + `src/lib/supabase.ts`).
- `@typescript-eslint/consistent-type-imports`: `prefer: 'type-imports'`, `fixStyle: 'inline-type-imports'`. Use `import type` or `import { type X }`.
- `@typescript-eslint/no-unused-vars`: unused arg/catch/destructure names must be prefixed with `_`.
- `no-duplicate-imports: error`.
- Excludes: `src/components/ui/` from ESLint.

## Env / backend

Copy `.env.example` → `.env`. Vars:

- `VITE_CLERK_PUBLISHABLE_KEY` — Clerk is **optional**. The app runs without it (shows a setup page at `/clerk`). Clerk integration lives in `src/routes/clerk/` + the provider in `src/main.tsx`. Can be fully removed.
- `VITE_SUPABASE_URL`, `VITE_SUPABASE_ANON_KEY` — client in `src/lib/supabase.ts`, also accepts `NEXT_PUBLIC_*` fallbacks. DB schema in `supabase/schema.sql` must be applied manually via Supabase SQL editor.
- `CLERK_SECRET_KEY` — used by dev API proxy (`/api/clerk`).
- `HOSTINGER_MAIL_TOKEN` — used by dev API proxy (`/api/mail`).

Local dev has proxy middleware: `/api/mail` → `api.mail.hostinger.com`, `/api/clerk` → `api.clerk.com/v1` (`vite.config.ts`).

## Deploy

- Netlify: `netlify.toml` redirects `/*` → `/index.html` (status 200).
- Vercel: `vercel.json` — framework `vite`, rewrites `/api/*` to `/api/$1`, everything else to `/index.html`.
- Both deploy with `pnpm build` → `dist/` directory.
